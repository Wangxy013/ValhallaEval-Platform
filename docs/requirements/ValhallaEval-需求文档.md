# 大模型Prompt与模型评测系统需求文档

## 一、文档概述

### 1.1 文档目的

本文档基于两次会议沟通的核心需求，完整梳理大模型Prompt与模型评测系统的全部业务要求、功能细节与落地规则，作为后续系统开发、测试、交付的唯一依据，同时留存会议核心决策要点，保障开发方向与需求方诉求完全一致。

### 1.2 项目背景

为解决大模型Prompt调优效果验证、多模型横向评测的效率问题，需搭建一套Web端评测系统，支持多版本Prompt、多模型的效果对比，提供灵活的校验与评估能力，降低Prompt调优与模型选型的成本，保障指令生效的可验证性。

## 二、项目核心目标

1. 实现多版本Prompt的效果对比，直观验证Prompt修改后的指令生效情况

2. 支持多模型的自由切换与横向评测，完成不同模型的效果差异对比

3. 提供人工+模型自动+自定义的多模式评测能力，满足不同场景的校验需求

4. 支持单条/批量测试数据的推理与结果汇总，保障测试的灵活性与一致性

5. 可视化展示评测结果，以直观的形式呈现校验、评估的全量信息

## 三、核心业务场景

1. **Prompt调优验证场景**：针对同一任务，对比修改前后的至少2版Prompt的推理结果，验证新增/修改的指令是否生效，定位Prompt的优化点

2. **模型横向评测场景**：使用同一套Prompt与测试数据，对比不同大模型的输出效果，完成模型选型的量化评估

3. **批量效果验证场景**：针对固定的测试数据集，批量完成多版本Prompt/多模型的推理，汇总整体效果，验证调优后的通用适配性

## 四、详细功能需求

### 4.1 系统核心流程

新建评测任务 → 选择评测对象（Prompt/模型）→ 配置测试数据 → 设置校验与评估规则 → 执行单条/批量测试 → 查看结果总览与明细 → 完成效果验证

### 4.2 分模块功能详情

#### 1. 模型管理模块

- 支持多主流大模型的接入与配置，当前阶段优先适配**豆包大模型**（用于功能验证），正式上线后核心适配**星火教育大模型**

- 预留模型扩展接口，支持后续新增模型的快速接入，用户可在任务中自由切换使用的模型

- 支持在同一评测任务中，同时选择多个模型进行并行测试，对比效果差异

#### 2. Prompt管理模块

- 支持多版本Prompt的录入、保存、版本命名与管理，支持对Prompt的修改点进行标注，方便对比不同版本的差异

- 支持在同一任务中，同时选择至少2个版本的Prompt进行并行测试，直观呈现修改前后的效果差异

- 支持Prompt的复用，可从历史任务中快速导入已有的Prompt版本

#### 3. 测试数据管理模块

- 支持测试数据集的手动录入、批量导入与管理，明确**测试数据固定原则**：任务启动后测试数据不可随意修改，保障评测的一致性与可复现性

- 支持单条测试数据、多条批量测试数据的灵活选择，适配单次验证与批量验证的不同需求

- 不依赖第三方Prompt Pilot工具，自主实现测试数据的管理、批量推理与结果爬取汇总能力

#### 4. 评测执行模块

- **单条测试能力**：支持单条测试数据执行1次或多次重复测试，验证结果的稳定性

- **批量测试能力**：针对多条测试数据批量执行推理，批量推理全程使用**全新无上下文会话**，不携带历史对话信息

- **核心执行规则**：优先支持单轮Prompt测试，多轮Prompt按单轮逻辑处理，不携带历史上下文

- 支持测试任务的暂停、继续与重新执行，保障批量任务的稳定性

#### 5. 评测校验模块

- 支持人工自定义填写校验点与校验标准，系统调用大模型按照校验标准，对推理结果进行一致性校验

- 校验结果支持对目标文本进行标记，以**文档批注的形式**直观展示校验情况，清晰呈现符合/不符合校验标准的内容

- 支持多校验点的批量配置，同一任务可设置多个校验维度

#### 6. 评测评估模块

内置三种预设评估模式，用户可按需选择，不可删减：

1. **人工效果评估**：提供评估入口，由人工对推理结果进行打分、标注评价，留存人工评估记录

2. **模型自动评估**：由大模型按照预设的通用评测标准，对结果进行自动打分、差异分析与评价

3. **自定义评估**：支持用户自定义评估规则、打分维度与要求，系统按自定义规则完成评估

#### 7. 任务管理模块

- **任务列表页**：展示所有创建的评测任务，支持任务的查看、编辑、删除、筛选操作

- **新建任务**：采用**分步骤引导**的交互模式，引导用户完成测试对象选择、测试数据配置、校验与评估规则设置、执行测试的全流程

- **任务结果页**：分为总览页与明细页：

    - 总览页：展示本次任务的汇总统计、多版本Prompt/多模型的整体效果对比、通过率/评分汇总

    - 明细页：展示单条数据的详细推理结果、校验批注情况、评估详情，支持单条结果的对比查看

## 五、界面需求

1. 系统为Web端产品，整体操作逻辑简洁，降低用户使用门槛

2. 核心页面包含：任务列表页、新建任务分步引导页、任务结果总览页、任务结果明细页

3. 结果对比页需支持左右分栏对比，同时展示多版Prompt、多模型的推理结果，直观呈现差异

## 六、非功能需求

1. **扩展性**：支持模型、评估规则、校验标准的灵活扩展，适配后续新增的业务需求

2. **稳定性**：批量测试任务执行过程中，支持断点续传，避免结果丢失；保障大模型调用的稳定性与异常重试能力

3. **数据一致性**：测试数据、Prompt版本、评测规则在任务启动后锁定，不可随意修改，保证评测结果的可复现性

4. **易用性**：操作流程符合用户的Prompt调优习惯，新建任务的分步引导清晰，无冗余操作

本部分为会议中明确的不可调整的核心规则，作为开发过程中的校验依据：

8. 第三方依赖：不使用Prompt Pilot工具，自主实现批量推理、结果爬取与汇总能力

9. 评测对象：支持两种评测对象，分别是Prompt（多版本对比）、大模型（多模型横向对比）
