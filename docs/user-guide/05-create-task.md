# 第5章 创建评测任务

## 概述

评测任务是一次完整的评测实验。通过 5 步引导向导完成配置，系统将自动规划推理计划并在执行时批量调用 LLM。

---

## 5 步创建向导详解

### 第1步：基本信息

| 字段 | 说明 |
|---|---|
| **任务名称** | 建议包含评测目的，便于日后查找。如 `答题助手格式优化 v1vsv2` |
| **任务描述** | 可选，记录本次评测的背景和目标 |
| **评测类型** | 决定任务的对比维度（见下方说明） |

**评测类型选择指南**

| 我想做... | 选择 |
|---|---|
| 验证 Prompt 修改是否生效，使用同一模型 | **Prompt 对比测试** |
| 对比不同模型的效果，使用同一 Prompt | **模型横向对比** |

---

### 第2步：选择模型

根据评测类型不同，有不同约束：

| 评测类型 | 模型数量要求 |
|---|---|
| Prompt 对比测试 | **恰好 1 个**（用于执行所有 Prompt 版本的推理） |
| 模型横向对比 | **2 个或以上**（每个模型各执行一遍同一 Prompt） |

> 勾选模型时，列表会显示模型名称、提供商和模型 ID，便于区分。

---

### 第3步：选择 Prompt

| 评测类型 | Prompt 数量要求 |
|---|---|
| Prompt 对比测试 | **2 个或以上**（通常是优化前后的版本） |
| 模型横向对比 | **恰好 1 个**（作为统一的测试 Prompt） |

> 建议选择名称相同、版本不同的 Prompt，这样对比结果更有参考价值。

---

### 第4步：测试数据

**选择数据集**：从下拉列表中选择已创建的数据集。

**选择具体数据条目**：

- 点击「全选」快速选中所有数据
- 也可以手动勾选部分数据（例如只选代表性题目做快速验证）
- 已选数量显示在表格上方

**重复执行次数**：

每条数据重复执行的次数（默认 1 次，最大建议 5 次）。

| 场景 | 推荐设置 |
|---|---|
| 验证指令是否生效（确定性任务） | 1 次 |
| 验证输出稳定性（生成型任务） | 2-3 次 |

> **重要**：选中的测试数据在任务创建时立即快照，之后对数据集的修改不影响本任务。

---

### 第5步：校验与评估

这一步是系统的核心差异化功能，决定了推理完成后如何客观衡量效果。

#### 校验检查点（Validation Checkpoints）

校验检查点是你对模型输出的**期望标准**，系统会在推理完成后自动用 LLM 裁判逐条判断是否 通过/未通过。

**如何设置有效的校验检查点：**

| 字段 | 说明 | 示例 |
|---|---|---|
| **检查点名称** | 简短的标准名称 | `格式规范` |
| **校验标准** | 具体的、可判断的描述 | `输出必须同时包含【答案】和【解析】两个标签，缺少任一则不通过` |

**好的校验标准应该：**
- 描述具体、可客观判断（避免主观词如"良好"、"清晰"）
- 一个检查点只描述一个维度
- LLM 裁判能根据描述明确判断 通过/未通过

```
✓ 好的校验标准：
  "输出必须包含【答案】和【解析】两个部分，缺少任一则不通过"
  "不能包含任何 HTML 标签"
  "解析部分字数不少于 50 字"

✗ 不好的校验标准（模糊）：
  "格式好看"
  "内容质量高"
```

可以添加多个检查点，覆盖不同维度（格式、内容准确性、完整性等）。

#### 评估模式

| 模式 | 说明 | 适用场景 |
|---|---|---|
| **人工评估** | 推理完成后手动为每条结果打分 | 需要人工判断的主观性评测 |
| **自动评估** | 系统调用 LLM 自动打 0-10 分（若有校验点则按标准逐项评价） | 大批量数据，快速获取量化评分 |
| **自定义规则** | 填写自定义评估规则（预留功能） | 有特定评分维度需求时 |

---

### 预计运行数

配置预览中会显示 **预计运行数**：

```
运行数 = 选中的模型数 × 选中的 Prompt 数 × 选中的测试数据数 × 重复次数
```

例如：1 个模型 × 2 个 Prompt × 10 条数据 × 1 次 = **20 次推理**

---

## 创建后的状态

任务创建后状态为 **「草稿」**，系统自动跳转到任务结果页。此时：
- 可以查看任务配置
- 可以修改配置（此时尚未执行推理）
- 点击「开始执行」开始批量推理

详见 [第6章 执行与校验](06-run-and-validate.md)。
