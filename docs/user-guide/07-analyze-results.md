# 第7章 分析结果

## 概述

任务结果页分为两个标签：**总览** 和 **明细**。总览提供汇总性的对比统计，明细展示每条测试数据的详细输出和差异分析。

---

## 总览页解读

### 核心统计卡片

| 指标 | 说明 | 何时有值 |
|---|---|---|
| **总运行数** | 全部推理次数 | 任务创建后即有 |
| **已完成** | 推理成功的次数 | 执行完成后 |
| **整体通过率** | 所有检查点的综合通过率 | 运行校验后 |
| **平均评分** | 所有推理的平均分（0-10） | 运行自动评估后 |

> 整体通过率 = 全部校验结果中通过数 ÷ 总校验数，跨越所有验证点和所有版本。单个验证点的详情见下方矩阵。

---

### 验证点对比矩阵（核心功能）

这是分析 Prompt 优化效果或模型对比的主要视图。

**矩阵结构：**

```
               │ 答题助手 (v1) [基准]  │ 答题助手 (v2)     │
───────────────┼──────────────────────┼──────────────────┤
格式规范       │ ████ 70%             │ ██████ 100% ↑+30% │
内容准确性     │ █████ 80%            │ █████ 90%  ↑+10%  │
知识点标注     │ ██ 40%               │ ████ 70%   ↑+30%  │
```

**如何解读：**

- **行** = 你设置的每个校验检查点（验证维度）
- **列** = 不同的 Prompt 版本或模型，其中第一个为**基准版本**（金色「基准」徽章）
- **进度条** = 通过率（绿色≥80%，黄色50-79%，红色<50%）
- **`+N%`/`-N%` 标签** = 与基准版本相比的变化量
  - `↑ +30% 改善` → 该检查点通过率提升了 30 个百分点
  - `↓ -10% 退步` → 该检查点通过率下降了 10 个百分点
  - `持平` → 通过率没有变化

**常见解读场景：**

1. **全面改善**：所有检查点都显示绿色 `↑`，说明 Prompt 优化有效
2. **部分改善**：某些检查点改善，某些退步，需要权衡哪个更重要
3. **关键维度退步**：某个重要检查点（如内容准确性）退步，即使总分提高也需谨慎

> **提示**：矩阵只在校验完成后才会显示通过率。校验未运行时，将显示空状态提示。

## 明细页解读

### 分组展示

明细页按**测试数据**分组，每个分组代表一道题/一条输入，展开后可看到该输入在不同 Prompt/模型下的所有输出。

**分组行信息：**
- 序号（与任务中锁定的测试数据顺序一致）
- 测试数据内容（截断显示）
- 版本数（如「2 个」）
- 完成状态
- 校验概况（通过/未通过数量）+ 有无差异

**展开后的运行卡片：**

每个版本/模型对应一张卡片，从左到右排列，**基准版本始终排在第一位**：
- **卡片头部**：Prompt/模型标签 + 运行状态 + 耗时 + 「基准」金色徽章（若为基准版本）
- **输入消息**（可折叠）：发送给模型的完整消息（系统提示 + 用户输入），支持点击「查看详情」弹窗查看全文
- **模型输出**：LLM 的完整回复（如有验证批注则高亮显示），支持点击「查看详情」弹窗查看全文
- **校验结果**：每个检查点的 通过/未通过 判断 + 标准文字 + 裁判说明，展示顺序与任务中定义的检查点顺序一致
- **评估结果**：综合评分 + 评价 + 各检查点子评分（如有）

### 明细页交互增强

- 点击测试数据内容，可弹窗查看完整题目
- 点击「全部展开 / 全部收起」，可一次性展开或收起全部测试数据分组
- 页面右下角提供「回到顶部」悬浮按钮，方便长结果页快速返回
- Prompt 对比场景下，版本卡片顺序固定为：**基准版本在最左侧，其余版本按 Prompt 版本号或更新时间排序**

---

### 对比差异分析

每个测试数据展开后，运行卡片下方会自动显示 **「对比差异」** 区域：

```
对比差异（以基准 v1 为参照）

↑ 改善  格式规范  — 基准 v1: 未通过 → v2: 通过
↓ 退步  内容准确性 — 基准 v1: 通过 → v2: 未通过
一致：知识点标注（均通过）
```

**差异标签说明：**

| 标签 | 含义 |
|---|---|
| `↑ 改善` | 该检查点至少有一个版本从「未通过」变为「通过」 |
| `↓ 退步` | 该检查点至少有一个版本从「通过」变为「未通过」 |
| `一致` | 所有版本该检查点结果相同（均通过或均未通过） |

---

### 筛选功能

明细页顶部提供筛选器：
- **按 Prompt 筛选**：只看特定版本的数据，选项展示 Prompt 名称/版本而不是内部 ID
- **按模型筛选**：只看特定模型的数据，选项展示模型名称而不是内部 ID

筛选后，分组和对比差异分析自动更新。

---

### 手动对比查看

如果想深度对比某两条输出（如模型对比场景下两个模型的输出），可以：
1. 切换到「单版本」视图（筛选出单个 Prompt 或模型）
2. 勾选两条运行记录
3. 点击「对比查看」打开双栏弹窗

## 结果分析建议

### Prompt 调优分析

1. **先看矩阵**：找到有差异的检查点
2. **关注退步项**：新版本是否在某个维度变差了？
3. **深入明细**：找几条「改善」或「退步」的数据，展开对比实际输出，理解模型的变化原因
4. **看平均分变化**：综合评分是否提升？

### 模型选型分析

1. **矩阵对比各模型通过率**：哪个模型在关键检查点更优秀
2. **查看平均评分**：综合质量对比
3. **关注失败样本**：某个模型是否在特定类型数据上系统性失败
4. **结合耗时和 token 消耗**：在质量相近时，考虑成本因素（明细页有耗时展示）
