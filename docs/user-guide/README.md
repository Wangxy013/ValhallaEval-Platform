# 用户使用手册

欢迎使用**大模型 Prompt 与模型评测系统**。

本系统专为 Prompt 调优验证和模型横向评测设计，帮助你快速、客观地衡量 Prompt 修改效果和不同模型的输出差异。

---

## 核心概念

在开始使用前，了解以下核心概念有助于快速上手：

| 概念 | 说明 |
|---|---|
| **模型配置** | 接入的大模型（如豆包、星火），包含 API Key 和连接参数 |
| **Prompt** | 系统提示词，可管理多个版本，每个版本代表一次调优迭代 |
| **测试数据集** | 固定的输入问题集合，评测时作为用户消息发送给模型 |
| **评测任务** | 一次完整的评测实验，绑定特定模型、Prompt、测试数据 |
| **评测类型** | `Prompt 对比测试`（同模型多 Prompt）或 `模型横向对比`（同 Prompt 多模型） |
| **校验检查点** | 用户自定义的评测标准，由系统调用 LLM 自动判断每条输出是否通过 |
| **验证校验** | 按校验检查点逐条判断 通过/未通过 的过程 |
| **评估** | 对推理结果打分（0-10），分人工、自动、自定义三种模式 |

---

## 推荐工作流

```
1. 配置模型          → 模型管理页面接入大模型 API
        ↓
2. 录入 Prompt       → 保存调优前后的多个版本
        ↓
3. 准备测试数据       → 创建数据集，录入代表性测试用例
        ↓
4. 创建评测任务       → 5步向导，选择对比对象、数据、设置校验标准
        ↓
5. 执行推理          → 点击"开始执行"，系统自动批量调用 LLM
        ↓
6. 运行校验          → 点击"运行校验"，系统自动判断每条输出是否符合标准
        ↓
7. （可选）自动评估   → 点击"自动评估"，LLM 对每条输出打综合分
        ↓
8. 查看分析结果       → 总览对比矩阵，明细差异高亮
```
