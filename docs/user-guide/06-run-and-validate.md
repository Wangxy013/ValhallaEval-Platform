# 第6章 执行与校验

## 概述

任务创建后，经历三个主要阶段：**推理执行** → **验证校验** → **评估打分**。本章详细介绍每个阶段的操作和注意事项。

---

## 阶段一：推理执行

### 开始执行

在任务总览页点击 **「开始执行」** 按钮。

系统会立即开始批量推理，你会看到：
- 任务状态变为 **「运行中」**
- 页面每 3 秒自动刷新，显示最新进度
- 总览统计中「已完成」数量持续增加

### 推理进度查看

切换到「**明细**」标签页可实时查看各条数据的推理状态：

| 状态 | 说明 |
|---|---|
| 待执行 | 等待轮到这条数据 |
| 运行中 | 正在调用 LLM |
| 已完成 | 推理成功，`output_content` 已填充 |
| 失败 | 调用出错（超时、API 错误等），`error_message` 有记录 |

### 暂停与恢复

- **暂停**：点击「暂停」，正在进行的 LLM 调用会等待完成，新的不再发起
- **恢复**：点击「继续执行」，从暂停位置继续

> 暂停不会丢失已完成的推理结果。

### 执行规则

- 每次推理使用**全新无上下文会话**（不携带历史对话）
- Prompt 作为 `system` 消息，测试数据作为 `user` 消息
- 超时时间 120 秒，超时视为失败

---

## 阶段二：验证校验

> **前提**：任务创建时需配置至少 1 个校验检查点。

### 触发校验

推理完成后，点击 **「运行校验」** 按钮。

系统会启动异步校验任务：
- 对每条 `已完成` 的推理结果 × 每个校验检查点，调用 LLM 裁判
- LLM 裁判会结合「校验点名称 + 校验标准 + 模型输出」进行判断，并用**中文**返回 `通过` 或 `未通过` 及简短说明
- 已校验过的配对自动跳过（幂等，可安全重复点击）

### 校验耗时

校验时间 ≈ `完成推理数 × 校验检查点数 × 单次 LLM 调用时间`

例如：20 条推理 × 2 个检查点 × 约 3 秒/次 = 约 120 秒

### 校验结果查看

校验完成后，在「**明细**」标签页展开任意测试数据：

- 每个运行卡片的「校验结果」区域显示各检查点的判断
- `通过` → 绿色背景，`未通过` → 红色背景
- 每条结果下方显示 LLM 裁判的简短说明

---

## 阶段三：评估打分

### 自动评估

点击 **「自动评估」** 按钮，系统调用 LLM 对每条推理结果打 0-10 分。

- 自动评估使用任务创建时指定的**评估阶段模型**
- 校验使用任务创建时指定的**校验阶段模型**
- 任务总览顶部会显示当前任务绑定的校验模型和评估模型，便于回溯

- 若配置了校验检查点，评分 prompt 会包含验证标准，LLM 会对每个标准单独评分（`checkpoint_scores`），再给出综合分
- 已评估过的推理记录自动跳过

评分参考标准：

| 分数段 | 含义 |
|---|---|
| 9-10 | 完全符合要求，质量优秀 |
| 7-8 | 基本符合要求，有小瑕疵 |
| 5-6 | 部分符合要求，有明显不足 |
| 0-4 | 未能满足基本要求 |

### 人工评估

如果评估模式设为「人工评估」，可在明细页展开每条记录，手动填写评分和评价（功能入口在运行卡片的评估区域）。

## 常见问题

**Q：点击"运行校验"报错"Task has no validation checkpoints"**

任务创建时未添加校验检查点。校验功能需要检查点，请重新创建任务并在第 5 步「校验与评估」中添加检查点。

**Q：推理有部分失败，能否只重跑失败的？**

当前版本暂不支持选择性重跑。可以点击「开始执行」重新执行整个任务（已完成的 run 会被重新推理并覆盖）。

**Q：校验结果感觉不对，裁判判断有误**

LLM 裁判并不完美。可以：
1. 完善校验标准，使描述更具体、更客观可判断
2. 在明细页手动查看输出，判断裁判是否合理
3. 重新运行校验（幂等，会跳过已校验的记录）
