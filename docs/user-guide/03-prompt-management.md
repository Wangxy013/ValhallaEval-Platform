# 第3章 Prompt 管理

## 概述

Prompt 管理页面用于录入和维护系统提示词的多个版本。每个 Prompt 版本代表一次调优迭代，通过版本号和修改说明记录演进历史。

---

## 创建 Prompt

1. 进入「Prompt 管理」页面
2. 点击「新建 Prompt」
3. 填写信息：

| 字段 | 说明 | 示例 |
|---|---|---|
| **名称** | Prompt 的功能名称（相同名称视为同一 Prompt 的不同版本） | `答题助手` |
| **版本** | 版本标识，自由格式 | `v1`、`v2`、`2024-03-20` |
| **内容** | 完整的系统提示词文本 | `你是一名专业的数学教师...` |
| **修改说明** | 本次修改了什么，方便后续对比 | `增加格式约束，要求输出答案和解析` |

---

## Prompt 版本管理最佳实践

### 命名建议

保持同一功能的 Prompt 名称一致，使用版本号区分迭代：

```
答题助手 v1  →  答题助手 v2  →  答题助手 v3
（基础版）       （加格式约束）    （加长度限制）
```

### 修改说明填写规范

好的修改说明能帮助你快速理解每次迭代的目的：

```
✓ 好的写法：
  "在输出末尾增加【知识点】标签，要求模型标注涉及的知识点"
  "删除了示例输出，缩短 prompt 长度以节省 token"

✗ 不好的写法：
  "修改了一些内容"
  "v2"
```

### 多版本并行测试

在创建 Prompt 对比任务时，可同时选择 2 个及以上版本。系统会用相同的测试数据分别测试每个版本，结果并排对比。

---

## 在评测任务中使用 Prompt

Prompt 在创建评测任务时关联，支持：
- **Prompt 对比任务**：选择 2+ 个版本（通常是优化前和优化后），使用同一个模型测试
- **模型对比任务**：选择 1 个版本，使用 2+ 个模型测试

---

## 注意事项

- Prompt 内容作为 `system` 消息发送给模型，测试数据作为 `user` 消息
- 系统采用**单轮推理**模式：每次推理使用全新会话，不携带历史对话
- Prompt 一经关联到已开始执行的任务，其内容即被快照锁定，后续修改不影响历史任务结果
