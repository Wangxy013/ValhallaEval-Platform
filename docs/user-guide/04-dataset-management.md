# 第4章 测试数据管理

## 概述

测试数据集是评测系统的"试卷"。每个数据集包含若干测试数据条目，每条条目作为用户消息发送给模型进行推理。

---

## 创建数据集

1. 进入「测试数据」页面
2. 点击「新建数据集」
3. 填写名称（如 `数学选择题 - 初中`）和可选描述
4. 保存后点击数据集名称进入详情页

---

## 添加测试数据

进入数据集详情页后，点击「添加测试数据」：

| 字段 | 说明 |
|---|---|
| **内容** | 发送给模型的用户消息，即测试输入 |
| **排序** | 数据项显示和执行的顺序（可选，默认按添加顺序） |

### 批量添加建议

如果有大量测试数据，可以：
1. 先在文本编辑器中准备好所有测试题
2. 逐条粘贴到「内容」输入框
3. 利用「全选」功能在创建任务时快速选取全部数据

---

## 测试数据设计原则

### 代表性

选择能覆盖目标场景主要情况的数据：

```
✓ 好的测试数据：
  - 包含各种难度级别的问题
  - 覆盖目标知识点的不同类型
  - 包含边界情况（如空输入、很长的输入）

✗ 应避免：
  - 所有数据都是同一类简单问题
  - 测试数据与实际使用场景差异过大
```

### 数量建议

| 场景 | 建议数量 |
|---|---|
| 快速验证 Prompt 改动 | 5-10 条 |
| 正式效果对比 | 20-50 条 |
| 全面质量评估 | 50-100+ 条 |

### 固定原则

**测试数据一旦在任务中使用，内容即被快照锁定**，后续对数据集的修改不影响已有任务。这保证了不同时间点的评测结果可以进行横向比较。

---

## 删除数据

- 删除单条数据：点击数据条目旁的「删除」图标
- 删除整个数据集：在数据集列表页点击「删除」（会同时删除所有数据条目）

> 注意：删除数据集不会影响已引用该数据集的历史任务，因为任务执行时已将数据内容快照保存。
