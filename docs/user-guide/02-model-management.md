# 第2章 模型管理

## 概述

模型管理页面用于配置评测系统可以调用的大模型 API。系统支持所有兼容 OpenAI Chat Completions 接口格式的模型。

---

## 支持的模型提供商

| 提供商 | 提供商标识 | API URL 示例 |
|---|---|---|
| 字节跳动豆包 | `doubao` | `https://ark.cn-beijing.volces.com/api/v3` |
| 科大讯飞星火 | `spark` | 由讯飞提供 |
| OpenAI | `openai` | `https://api.openai.com/v1` |
| 其他 OpenAI 兼容服务 | `openai_compatible` | 由服务商提供 |

> 系统底层使用 OpenAI Chat Completions 格式（`POST {api_url}/chat/completions`），任何兼容该格式的模型均可接入。

---

## 新建模型配置

1. 进入「模型管理」页面
2. 点击「新建模型」按钮
3. 填写配置信息：

| 字段 | 说明 | 示例 |
|---|---|---|
| **模型名称** | 便于识别的自定义名称 | `豆包 Pro-32k` |
| **提供商** | 选择提供商类型 | `doubao` |
| **API URL** | 接口基础地址（不含 `/chat/completions`） | `https://ark.cn-beijing.volces.com/api/v3` |
| **API Key** | 鉴权密钥 | `sk-xxx` |
| **模型 ID** | 传给 API 的模型标识符 | `ep-20240101-xxxxx` |

4. 点击「保存」

---

## 验证模型是否可用

模型保存后，你可以通过创建一个简单任务（选择该模型，加入 1-2 条测试数据）来验证连通性。如果推理成功完成，说明配置正确。

---

## 管理现有模型

- **编辑**：点击模型卡片上的「编辑」按钮，可修改任意字段（API Key 更新时直接覆盖）
- **删除**：点击「删除」，注意：已在任务中使用的模型不建议删除，否则历史任务展示可能出现异常

---

## 注意事项

- API Key 以明文存储在数据库中，建议确保数据库访问权限受限
- 同一提供商可以配置多个模型（如豆包 Standard 和 Pro），在模型对比任务中可同时选择
- 系统不验证 API Key 有效性，配置后需通过实际调用确认
