# 系统化调试

## 适用场景

当测试失败、运行时行为异常、接口结果不符合预期，或问题来源尚不明确时，先使用该流程再修改代码。

## 必须遵循

1. 用最小可重复方式复现问题。
2. 记录实际症状、报错信息和触发条件。
3. 缩小问题范围到明确的层级、文件、查询或接口边界。
4. 在修改前识别根因，不用猜测性改动代替分析。
5. 采用最小修复，避免顺手扩大重构范围。
6. 重新运行复现路径和相关回归检查。

## 仓库要求

- 区分前端 helper 逻辑、后端 API、数据库状态和模型调用链路，不要混为一谈。
- 涉及任务创建、校验、评估或结果分析时，必须检查三阶段模型拆分是否仍然成立。
- 涉及模型配置时，必须同时关注 API Key 掩码行为与更新行为。
